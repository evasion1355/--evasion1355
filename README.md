## 最小可交付物（MVP）

1. **系统搭建**

   * 部署一套 PostgreSQL / openGauss。
   * 创建测试用户和角色，设置不同权限（SELECT、INSERT、UPDATE、DELETE 等）。
2. **权限扫描**

   * Python 脚本连接数据库。
   * 获取所有用户及其权限信息：

     ```sql
     SELECT grantee, privilege_type, table_name
     FROM information_schema.role_table_grants;
     ```
   * 标记高风险权限（如全库写入权限给非管理员用户）。
3. **审计记录分析**

   * 开启数据库审计日志（PostgreSQL: `pgaudit`，openGauss: `audit`）。
   * Python 脚本读取日志，统计：

     * 高权限操作的次数
     * 异常访问模式（如非工作时间访问）
4. **输出整改建议**

   * 根据扫描结果生成整改建议，例如：

     * “用户 A 拥有全库写权限，建议限制为只读或针对指定表授权。”
     * “检测到用户 B 在非工作时间频繁访问敏感表，建议审查访问原因。”
   * 输出为 Excel 或 Markdown 文件，包含风险等级（高/中/低）。
5. **可选Web展示**

   * Flask + 前端可展示权限分布和风险统计图表。
